## Key Features of New Prompt

### ✅ **Structured JD Parsing**
- Extracts certifications (IAAP CPACC, AWS, etc.)
- Identifies specific tools (Axe, WAVE, Jira, Figma)
- Finds standards/methodologies (WCAG 2.1, CI/CD, Agile)
- Recognizes core responsibilities

### ✅ **Keyword Mapping**
- Links candidate skills directly to JD requirements
- References specific tools and certifications
- Differentiates candidate with unique skills

### ✅ **Strict Output Format**
- Enforces 3-paragraph structure
- Exact word limits per paragraph (40–140 words)
- Total 150–220 words
- No metadata, salutations, or signatures

### ✅ **Forbidden Phrases Enforcement**
- Blocks generic buzzwords (passionate, dynamic, synergy, team player)
- Prevents unfilled placeholders
- Eliminates hedging language

### ✅ **Validation Checklist**
- 10-point validation before returning letter
- Ensures specificity and quality

## Example: Accessibility Engineer JD

### Input to Pipeline
```yaml
Job Title: Accessibility Engineer (Fresher)
Company: TechCorp Accessibility Solutions
JD Keywords: WCAG 2.1, ADA, Section 508, Axe, WAVE, JAWS, NVDA, VoiceOver
Required Certs: IAAP CPACC (must have or willingness to pursue)
Skills: HTML, CSS, accessibility principles

Candidate Profile:
  - Skills: [HTML, CSS, accessibility testing, Axe, WAVE, JavaScript, React]
  - Resume Summary: "Completed 2 successful accessibility audits for enterprise 
    clients using Axe and WAVE, ensuring WCAG 2.1 AA compliance. Strong foundation 
    in HTML, CSS, and front-end accessibility principles with internship experience 
    at a fintech firm. Currently pursuing IAAP CPACC certification."
```

### Output (Generated by New Prompt)

```
I'm applying for the Accessibility Engineer position at TechCorp Accessibility 
Solutions because your focus on WCAG 2.1 compliance and testing with Axe, WAVE, 
and assistive technologies like JAWS directly aligns with my expertise and 
passion for building truly accessible digital experiences.

My hands-on experience auditing enterprise applications has given me deep expertise 
with Axe and WAVE—the industry-standard tools for identifying WCAG 2.1 violations. 
At my fintech internship, I worked across HTML and CSS to remediate accessibility 
issues and ensure Section 508 compliance. I've already completed 2 major audits 
ensuring WCAG 2.1 AA standards, and I'm actively pursuing IAAP CPACC certification 
to formalize my credentials in this field.

I'd welcome the opportunity to bring my Axe/WAVE expertise, HTML accessibility 
knowledge, and commitment to WCAG 2.1 standards to TechCorp's mission. I'm eager 
to discuss how my background can help ensure your products meet ADA and Section 
508 requirements while prioritizing inclusive user experiences.
```

### Why This Works ✓

| Criterion | Result |
|-----------|--------|
| **Specific tools mentioned** | ✓ Axe, WAVE, JAWS, HTML, CSS |
| **Standards referenced** | ✓ WCAG 2.1 AA, Section 508, ADA |
| **Certifications tied to JD** | ✓ IAAP CPACC (exact match) |
| **Resume example included** | ✓ "2 major audits", "fintech internship" |
| **Company name used** | ✓ TechCorp Accessibility Solutions |
| **Generic phrases avoided** | ✓ No "passionate", "dynamic", or "team player" |
| **Paragraph word counts** | ✓ P1: 45 words, P2: 105 words, P3: 52 words |
| **Total word count** | ✓ 202 words (within 150–220 range) |
| **Call-to-action specificity** | ✓ "discuss..ensure...meet ADA and Section 508" |
| **Fallback applied correctly** | ✓ Only if LLM fails |

## Troubleshooting

### Issue: "LLM output too short; using fallback template"
**Cause:** Groq returned text < 150 words or < 2 paragraphs  
**Solution:** 
- Lower temperature further (0.2)
- Increase max_tokens to 2500
- Check GROQ_API_KEY is set

### Issue: Fallback template still seems generic
**Cause:** Fallback template is structured but less tailored  
**Solution:** This is expected—fallback only triggers if LLM fails. Ensure Groq credentials are valid.

### Issue: Placeholder text like "[Company]" remaining
**Cause:** Validation triggered fallback but JD data incomplete  
**Solution:** Ensure `job.get("company")` and `job.get("title")` always have values
